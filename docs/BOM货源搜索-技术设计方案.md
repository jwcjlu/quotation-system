# BOM 货源搜索与配单 — 技术设计方案

本文档在 [BOM单货源搜索与询价需求文档](./BOM单货源搜索与询价需求文档.md) 基础上，根据产品澄清结论固化**范围、边界、状态机与实现要点**，用于指导后端、Agent 与前端分工。

**澄清结论来源**：上传/解析后平台可选、合并任务、服务器时区、失败重试策略、轮询、进程内事件、固定映射、Agent 并发规则、仅比价导出但结果落库、待确认展示规则等（见下文 §2）。

---

## 1. 目标与范围

| 项 | 说明 |
|----|------|
| 业务目标 | BOM 导入 → 解析 → 按会话所选平台拉取/复用当日报价或「无该型号」终态 → 自动配单比价 → 用户可导出；**不配下单、不接支付**（第一期）。 |
| 持久化 | **配单结果需存储**，支持后续浏览历史配单/结果（与「历史 BOM」能力一致或扩展为独立「配单记录」实体）。 |
| 非目标（第一期） | 购物车闭环、锁价下单、实时推送（SSE/WebSocket）。 |

---

## 2. 已澄清的产品与架构决策

| # | 主题 | 决策 |
|---|------|------|
| 1 | **平台选择时机** | **上传时**可选平台；**解析完成后仍可调整**。每次有效集合以**当前 UI 勾选**为准；变更后须**重算就绪**并**增量**增删搜索任务（见 §5）。 |
| 2 | **同型号多行** | **合并**：同一 BOM 会话、同一型号、同一平台、同一业务日只对应 **一条**「型号×平台」搜索任务与一条缓存结论（与需求文档 **型号×平台×业务日** 主键一致）；多行共享该结论，行级差异仅体现在数量等 BOM 行字段。 |
| 3 | **业务日** | **服务器时区**下的自然日 0:00–24:00；`biz_date` 以服务器日期为准写入与查询。 |
| 4 | **失败语义** | **执行失败、超时、网络错误等不得记为「无该型号」**；须进入 **failed/可重试** 路径，**允许自动重试直至成功**，或**人工介入**（取消/改配/重试）后结束。仅当采集逻辑判定平台确实无该料时，记 **`no_mpn_match` 终态**。 |
| 5 | **重试与人工** | **自动重试**与 **手动重试** **分开计数**（手动为**额外计数**，不占自动重试额度）。自动重试：可配置次数/退避，耗尽后进入待人工；用户可 **手动重试**，每次手动独立累加 `manual_attempt`（与 `auto_attempt` 分离）。当日同一「型号×平台×日」仍视为同一业务意图，仅区分两类计数便于审计与限流。 |
| 6 | **前端就绪感知** | **仅轮询**（定时请求会话/型号就绪状态）；不做 SSE/WebSocket。 |
| 7 | **「广播」实现** | **进程内事件**（单机多协程/多模块解耦）；若未来多实例部署，再引入 Redis/NATS 等外置消息（当前不强制）。 |
| 8 | **平台 ↔ 脚本** | **固定映射**：每个平台 ID 对应固定 `script_id`（及版本策略），见 §7。 |
| 9 | **Agent 并发** | **同一物理 Agent** 可安装并执行**多个平台**脚本；**同一平台**在同一 Agent 上 **同时最多 1 个任务**（与架构「同 `script_id` 串行」一致：通常 **一平台一脚本包**，即同平台任务自然串行）。不同平台任务可并行（受 Agent 资源与实现限制）。 |
| 10 | **交付物** | **比价 + 导出**；配单/比价**结果落库**可追溯。 |
| 11 | **「待确认」展示** | 除行级状态外，**展示哪些所选平台尚无有效数据**，并给出**原因**（未执行 / 执行中 / 失败可重试 / 已判无料等——字段级见 §6）。 |

**说明**：早期选择题「B」与现澄清「上传可选 + 解析后可调」兼容：以**最终勾选集合**驱动任务，**解析后调整**为必支持能力。

## 4. 核心状态机（简述）

### 4.1 型号 × 平台 × 业务日（搜索侧）

| 状态 | 含义 |
|------|------|
| `missing` | 尚无记录 |
| `pending` | 已建任务，待派发 |
| `dispatched` | 已下发 Agent |
| `running` | 执行中 |
| `succeeded_quotes` | 终态：有报价（`has_quotes`） |
| `succeeded_no_mpn` | 终态：无该型号（`no_mpn_match`，**仅真实空结果**） |
| `failed` | 非终态失败，**可自动重试**或**人工处理** |
| `cancelled` | 用户取消或会话废弃（产品规则） |

**规则**：从 `failed` **不得**直接写入 `succeeded_no_mpn`；必须重试成功得到平台真实响应，或由人工关闭任务。

**重试计数（落实 §2 #5）**

| 字段（建议） | 说明 |
|--------------|------|
| `auto_attempt` | 自动重试已执行次数，上限由配置决定 |
| `manual_attempt` | **手动重试次数，额外计数**，与 `auto_attempt` 互不影响 |

### 4.2 BOM 行（配单侧）

沿用需求 **完全匹配 / 待确认 / 无法匹配**，但 **待确认** 时 UI **必须**展示：所选平台中仍无有效结论的平台列表及原因（见 §6）。

## 5. 平台勾选变更（解析后可调）

1. 维护 `platform_selection_revision`（或等价时间戳）：每次用户**确认勾选变更**时递增。  
2. **重算**：对新旧集合做差集：  
   - **新增平台**：对会话内所有型号补建缺失的「型号×平台×biz_date」任务（若缓存无终态）。  
   - **移除平台**：取消**仅属于被移除平台**且仍为 `pending` 的任务；已写入缓存的保留供其他会话复用，但**当前会话**配单不再引用被移除平台。  
3. **合并**：同型号多行不重复建任务（§2 表格 #2）。

## 6. 轮询 API 建议（面向前端）

为满足 **仅轮询** 与 **待确认原因**，建议提供：

| 接口（示例） | 用途 |
|--------------|------|
| `GET /api/v1/bom-sessions/{id}/readiness` | 返回会话级：整体进度、是否可进入配单、**阻塞原因摘要** |
| `GET /api/v1/bom-sessions/{id}/lines` 或扩展字段 | 每行：`match_status`；当 `待确认` 时附带 `platform_gaps[]: { platform_id, phase, reason_code, message }` |

`reason_code` 示例：`NO_RECORD` / `PENDING` / `RUNNING` / `FAILED_RETRY` / `FAILED_MANUAL` / `NO_MPN`（仅终态无料）。

## 7. 平台 ID 与 `script_id` 固定映射（建议稿）

实现时以配置或数据库配置表为准；以下为与需求文档 **§1.3** 对齐的**逻辑固定映射**：

| platform_id | script_id（建议） | 备注 |
|-------------|-------------------|------|
| `find_chips` | `find_chips`（与 ZIP / `version.txt` 一致） | 与仓库 `find_chips_crawler.py` 对应 |
| `hqchip` | `hqchip` | 与 `hqchip_crawler` 对齐 |
| `icgoo` | `icgoo` | |
| `ickey` | `ickey` | 云汉 |
| `szlcsc` | `szlcsc` | 立创 |

**版本**：任务载荷 `version` 与 Agent 本地脚本包 `version.txt` 一致；升级策略见《分布式采集 Agent》。

## 8. Agent 侧并发（落实澄清 §2 #9）

- **不同 `script_id`（不同平台）**：同一 Agent **可并行**多个任务（若实现允许进程/线程资源）。  
- **同一 `script_id`（同一平台）**：**串行**，同时最多 1 个任务（架构 §4.5 与用户需求一致）。  
- 调度器派单时：对 `(agent_id, platform/script_id)` 维度做 **互斥**，避免同平台重叠执行。

---

## 9. 数据持久化

| 实体 | 要点 |
|------|------|
| 搜索任务 | `mpn`, `platform_id`, `biz_date`, `state`, **`auto_attempt`**, **`manual_attempt`**（手动额外计数）, `last_error`, `session_id`（可选）, `selection_revision` |
| 当日报价缓存 | `outcome`, 报价列表或 `no_mpn_match` 证据字段 |
| **配单结果快照** | 会话 ID、行列表、推荐来源平台、单价、小计、策略版本、**结果版本号**、创建时间；支持历史列表与详情浏览 |

导出可基于快照生成文件（Excel/PDF 由产品定）。

---

## 10. 进程内事件

- 新任务入队、任务终态变更时，向**搜索任务模块**发布进程内事件，触发尽快尝试派发（无需外置 MQ）。  
- 多实例部署时：本方案**不保证**跨实例即时性；需后续替换为外置队列/订阅。

---

## 11. 风险与后续演进

| 风险 | 缓解 |
|------|------|
| 进程内事件无法跨实例 | 文档化；上线多实例前引入外置消息或 DB 轮询派发 |
| 「重试直至成功」可能长时间占用队列 | 配置自动重试上限 + 告警 + 人工入口；手动重试**单独计数**、可另配上限（可选） |
| 固定映射变更需发版 | `script_id` 配置表化 |
