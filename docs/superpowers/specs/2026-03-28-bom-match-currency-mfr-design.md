# 自动配单 — 币种归一与厂牌别名（设计决策）

> **定位：** 对 [2026-03-27-bom-sourcing-requirements.md](./2026-03-27-bom-sourcing-requirements.md) §6「配单」中 **价格口径** 与 **厂牌匹配** 的细化约定。  
> **输入数据：** 以 `t_bom_quote_cache`（主键 `id`）及其明细表 `t_bom_quote_item` 为准（原 `quotes_json` 已拆表），并结合各平台回传结构进行字段映射。

---

## 1. 币种：单机选择 + 读取时归一 + 双币展示 + 明确排序键

### 1.1 单机配置（部署级）

- 每个 **服务实例**（单机部署）配置一项 **「配单比较/展示基准币种」**（例如 `CNY` 或 `USD`），下文记为 **`base_ccy`**。
- 配单、列表、导出等 **排序与「最低价」比较** 一律在 **`base_ccy`** 下完成，避免同一请求混用多种口径。

### 1.2 读取时归一（Compare-time normalization）

- **不在入库时强制改写** 各平台原始报价（保留爬虫/Agent 回传的原始字段，便于审计与对账）。
- 在 **配单读取路径**（或等价的「展开缓存 → 构造候选报价」步骤）中：
  - 解析每条候选的 **标价币种**（来自结构化字段；若仅字符串阶梯价，见 §1.5）。
  - 使用 **配置的汇率表或汇率服务**（具体来源在实现/运维文档中固定：如每日中间价、手动配置表）将 **单价** 换算为 **`base_ccy`**，得到 **`unit_price_base`**。
- 汇率应用规则须 **可复现**：记录 **汇率版本标识**（日期 + 来源）在行级配单结果或审计字段中（实现阶段定最小集）。

### 1.3 双币展示

- UI/API 返回中，对参与排序的报价至少展示：
  - **原始金额 + 原始币种**（来自平台或解析结果）；
  - **换算后金额 + `base_ccy`**（即 `unit_price_base` 或与数量结合后的行金额）。
- 「双币」指 **原币与基准币**；若原币已与 `base_ccy` 相同，可只展示一列或重复标注同一币种（实现可选，行为须一致）。

### 1.4 明确排序键

- **自动择优的排序键** = **`unit_price_base`**（在 **同一数量阶梯规则** 下选取可比单价后，再换算到 `base_ccy` 的值）。
- **禁止** 在原币与 `base_ccy` 混用的情况下比较数值；**禁止** 未标注币种时按裸 `float` 排序。

### 1.5 阶梯价与缺失结构化单价

- 若结构化字段（§1.7 第 1–3 步）均无法得到 **`compare_unit_price`**，对 **`price_tiers`**（及映射表指定的等价字段）采用 **§1.11 字符串解析**。解析成功则进入 §1.2 换汇；**解析失败或未识别格式** 时：**不参与自动最低价排序**，标记为 **需人工**（保守回退）。
- 与需求文档 §6 **硬约束（库存 ≥ 数量）** 一致：选档所用 **Q** 为 BOM 需求数量，且 **库存 ≥ Q** 后才进入比价。
- **V1 默认：开启 §1.11 解析。** 若部署需整体关闭字符串解析（仅接受结构化单价），通过 **配置开关** 关闭；关闭后第 4 步跳过，直接落入保守回退。

### 1.6 比较用单价口径（税 / 费用）

- 自动比价使用的 **`compare_unit_price`（原币）** 定义为：**不含运费、不含关税的标价单价**；各平台字段映射到「税前/未税」或「含税」须在 **平台字段映射表**（实现维护，可与 proto 注释或单独表格同库）中 **逐平台写死**。
- 若某平台 **仅提供含税价**：映射表须注明 **换算为可比口径的规则**（例如除以 \(1+\)税率），或声明该平台 **不参与自动比价**（与 §1.5 **解析失败/关闭解析** 时的保守回退一致）。
- **单一事实源**：禁止各服务实例对同一字段自行解读；映射表变更须可审计。

### 1.7 多价格字段优先级

从单条报价结构（如 `PlatformQuote` / `t_bom_quote_item` 行）得到 **`compare_unit_price` 与原币 `quote_ccy`** 时，按 **自上而下首次成功** 为准：

1. **`unit_price` > 0** 且 **`quote_ccy` 可确定**（字段自带或映射表按 `platform_id` 默认）：采用之。  
2. 否则尝试 **`mainland_price`**：按 §1.9 在 **BOM 数量** 下解析出 **一档单价 + 币种**。  
3. 否则尝试 **`hk_price`**：同上。  
4. 否则按 §1.11 尝试 **`price_tiers`**（及映射表指向的同语义字段）**字符串解析**；若 §1.5 配置关闭字符串解析则跳过本步。  
5. 否则：该候选 **无 **`compare_unit_price`**，不参与自动最低价排序**（保守回退：需人工）。

### 1.8 汇率日期

- 查表所用 **汇率日期** = 该候选所属缓存行 **`bom_quote_cache.biz_date`**（日历日，与时区约定在实现中固定）。  
- 若该键无效或缺失：退化为 **配单请求当日**（UTC 或业务时区，二选一并写死），并在审计字段中标记 **`fx_date_source`** = `biz_date` | `request_day`。  
- **汇率版本标识** 至少包含：`fx_date`、`fx_table_version`（或服务商批次号）。

### 1.9 MOQ 与数量阶梯（选档语义）

- 设 BOM 需求数量为 **Q**（正整数；小数数量在实现中 **固定一种取整规则**，如向上取整到整数件，并写入映射表/实现说明）。  
- 若报价存在 **整体 MOQ** 且 **MOQ > Q**：该候选 **剔除**，不满足需求 §6 硬约束。  
- **阶梯价（含 §1.11 解析结果）**：在档位集合 \(\{(moq_i, price_i)\}\) 中，先取满足 **\(moq_i \le Q\)** 的所有档；若集合为空（**Q 小于任一档位起订量**），该候选 **剔除**。否则取 **\(moq_i\) 最大** 的一档，其 **\(price_i\)** 即为 **`compare_unit_price`**（即：对订购量 **Q** 生效的 **最高阶梯** 单价，与常见 B2B 阶梯价规则一致）。  
- 若结构化字段与字符串解析 **同时存在且结果冲突**：以 **§1.7 更靠前且已成功** 的步为准；**禁止** 混用两个来源。  
- 若无法 **唯一** 确定适用档（如同一 `moq` 重复且单价不同）：**解析失败**，走 §1.5 保守回退。

### 1.10 舍入与平局（次排序键）

- **换算与比较**：`unit_price_base` 在比较前须 **固定舍入规则** 二选一（全仓库一致）：  
  - **A**：换算到 `base_ccy` 后以 **最小货币单位整数**（如 CNY「分」）比较；或  
  - **B**：`Decimal` **保留 6 位十进制** 再比较。  
- **`unit_price_base` 平局**（按所选舍入规则相等）时，按以下 **确定性** 次序打破平局（前者优先）：  
  1. **交期（期货/货期）更短者优先**：从报价结构化字段（如 `lead_time`，具体字段名以 **平台字段映射表** 为准）解析出 **可比天数 `lead_days`**（非负整数或工作日起算规则在映射表 **按平台写死**）。  
     - **区间**（如「3-5 天」「3~5」）：取 **下限** 作为 `lead_days`（保守：更短交期优先）。  
     - **`N/A`、空、现货、当天发货** 等：在映射表定义 **固定数值**（如现货 = 0）或 **解析失败**。  
     - **解析失败** 或 **无字段**：`lead_days = +∞`（视为交期未知/最差，**排在有可解析交期的候选之后**）。  
     - 比较规则：**`lead_days` 升序**（数值小者优先）。  
  2. **`lead_days` 仍相同**：**可解析库存** 大者优先（`stock` 等字段，解析失败视为 0）。  
  3. **再平局**：**`platform_id` 字典序 ascending**。

### 1.11 `price_tiers` 字符串解析（V1，已启用）

本节描述当前 **已观测** 的三家平台样例共通的解析规则；**其它格式** 须 **解析失败 → 保守回退**，后续再在映射表按 `platform_id` 扩展分支。

#### 1.11.1 分隔与单档语法

- 整串按 **`|`** 拆成多段；每段 `trim` 后独立解析。  
- **单档** 模式（正则意图描述，实现可用等价代码）：  
  - **`^\s*(\d+)\+\s*([$￥¥])\s*(\d+(?:\.\d+)?)\s*$`**  
  - 组 1：**`moq`**（正整数，含义为「N 件起」）。  
  - 组 2：**币种符号** → `$` 映射 **`USD`**；`￥` 或 `¥`（全角半角）映射 **`CNY`**。  
  - 组 3：**单价**（十进制小数）。  
- **段内任一不符合**：整串 **解析失败**（保守回退），**禁止** 部分档位静默丢弃（避免比价偏差）。

#### 1.11.2 选档

- 对解析得到的列表按 §1.9 选取 **\(moq_i \le Q\)** 中 **\(moq_i\) 最大** 的一档，得到 **`compare_unit_price`** 与 **`quote_ccy`**。  
- 各平台样例中档位 **书写顺序可升可降**，算法 **不依赖** 原文顺序，仅依赖数值。

#### 1.11.3 平台样例（验收/单测建议直接引用）

| platform_id | 币别 | price_tiers 原文样例（节选） |
|-------------|------|------------------------------|
| find_chips | USD | `3000+ $4.1930 \| 1000+ $4.4418 \| 500+ $4.6114 \| 250+ $4.7914 \| 100+ $5.0464 \| 25+ $5.4720 \| 10+ $5.7820 \| 1+ $6.6700` |
| hqchip | CNY | `1+ ￥19.8235 \| 10+ ￥14.6639 \| 100+ ￥12.569 \| 1000+ ￥10.4742` |
| ickey | CNY | `1+ ￥14.0729 \| 10+ ￥12.7522 \| 30+ ￥10.5661 \| 100+ ￥10.2928 \| 300+ ￥9.7281 \| 1000+ ￥9.1087` |

**举例（Q=100，ickey）**：适用档为 `100+ ￥10.2928`，故 **`compare_unit_price=10.2928`，`quote_ccy=CNY`**。  
**举例（Q=500，find_chips）**：适用档为 `500+ $4.6114`，故 **`compare_unit_price=4.6114`，`quote_ccy=USD`**。

#### 1.11.4 与 `mainland_price` / `hk_price` 的关系

- 若 §1.7 第 2、3 步将 `mainland_price`/`hk_price` 定义为 **与 `price_tiers` 同形字符串**，则 **使用同一解析器**；若某平台该字段为 **其它格式**，须在映射表标注 **仅走 `price_tiers`** 或单独格式版本号，避免误解析。

## 2. 厂牌 / 制造商：维护别名映射

### 2.1 目标

- 在需求「型号一致 + BOM 已填字段完全匹配」前提下，**厂牌** 不采用裸字符串相等，而是 **先映射到规范名再比较**。

### 2.2 规范名与别名表

- 维护 **`manufacturer_alias`**（名称以 DDL 为准），至少支持：
  - **`canonical`**：规范厂牌名（或规范 ID + 展示名，实现二选一并全局一致）；
  - **`alias`**：可出现于 BOM 或报价中的写法（多行多别名指向同一 `canonical`）。
- **匹配流程**：
  1. 对 BOM 行厂牌字符串做基础规范化（trim、全半角、大小写等，规则与需求 §6「同一规范化规则」一致）。
  2. 用别名表解析：**若命中别名 → 替换为 `canonical`；若不命中 → 按产品策略处理**（见 §2.3）。
  3. 对报价侧厂牌字段做同样规范化 + 别名解析。
  4. 比较双方 **解析后的 `canonical`（或规范 ID）** 是否相等。

### 2.3 未命中别名时的策略（须在产品/实现中固定一种）

- **严格**：未命中则 **不匹配** 该行厂牌约束（该行不进入自动配单或标为失败）。  
- **宽松（可选后续版本）**：未命中时退化为规范化字符串相等 —— **不作为 V1 默认**，除非需求文档显式打开。

### 2.4 与型号、封装、参数的关系

- **型号（MPN）**：仍以 `mpn_norm` 与行级规范化规则为硬门槛，**不**通过厂牌别名表纠正型号。
- **封装、参数**：仍按需求 §6 **完全匹配**；别名表 **仅用于厂牌维度**。

### 2.5 BOM 行未填厂牌

- BOM **未填写厂牌**（空或仅空白）：**不施加厂牌约束**，厂牌维度 **视为通过**（仍须满足 MPN、封装、参数等其它约束）。

### 2.6 报价侧无厂牌

- BOM **已填厂牌** 而报价 **缺少厂牌字段或为空**：该候选 **厂牌不匹配**，**不参与** 自动配单。  
- BOM **未填厂牌**（§2.5）：报价无厂牌 **允许通过** 厂牌维度。

### 2.7 别名表数据约束与命中规则（V1）

- **唯一性**：**规范化后** 的 **`alias` 全局唯一**（禁止两行指向不同 `canonical`）；导入/编辑 **冲突则拒绝**。  
- **V1 命中规则**：规范化后的输入与表中某 **`alias`（规范化后）完全相等** 即命中对应 `canonical`。**不启用** 前缀/包含匹配，以降低误判；若未来启用，须另 spec 并配套歧义检测。  
- **建议数据模型**：**`canonical_id`（稳定主键）+ `display_name`**；比较用 `canonical_id`，展示与日志用 `display_name`。若 V1 仅字符串 `canonical`，须保证 **全局唯一且稳定**。

## 3. 横切约定（缓存、审计）

- **`outcome` 无成功报价、`t_bom_quote_item` 无明细或明细不可解析**：该平台 **不产生候选**，不参与比价。  
- **历史配单与重放**：建议在持久化结果中保留 **`unit_price_base`、`original_amount`、`original_ccy`、`fx_date`、`fx_date_source`、`compare_price_field`**（取自 §1.7 哪一步；若来自 §1.11 可记 `price_tiers_parsed`）等最小集，便于审计与复现当日排序。

### 3.1 `t_bom_quote_item` 建议字段（V1）

- 与缓存主表关系：`t_bom_quote_item.quote_id` → `t_bom_quote_cache.id`（一对多）。
- 建议保留平台原始字符串字段，解析在读取路径完成（与 §1.2 一致）。

| 字段 | 类型（建议） | 说明 |
|------|--------------|------|
| `id` | BIGINT UNSIGNED | 主键，自增 |
| `quote_id` | BIGINT UNSIGNED | 外键，关联 `t_bom_quote_cache.id` |
| `model` | VARCHAR(255) | 报价型号（平台返回） |
| `manufacturer` | VARCHAR(255) | 厂牌原文 |
| `stock` | VARCHAR(64) | 库存原文（允许 `N/A`） |
| `package` | VARCHAR(255) | 封装原文 |
| `desc` | TEXT | 描述原文 |
| `datasheet_url` | TEXT | 数据手册 URL |
| `moq` | VARCHAR(64) | 起订量原文（允许 `N/A`） |
| `lead_time` | VARCHAR(128) | 交期原文（如 `1工作日`） |
| `price_tiers` | TEXT | 阶梯价原文（如 `5+ ￥1.3624 | ...`） |
| `hk_price` | TEXT | 香港价原文 |
| `mainland_price` | TEXT | 大陆价原文 |
| `query_model` | VARCHAR(255) | 查询型号（用于回溯） |

> 示例（规范 JSON）：
> `{"id":1,"quote_id":111,"model":"TS5A3159DCKR","manufacturer":"Texas Instruments","stock":"43787","package":"SC-70-6","desc":"Analog Switch Single SPDT 6-Pin SC-70 Tape/Reel","datasheet_url":"https://pf01.ickimg.com/datasheet/23/93/b242/23/960b8eab58f119ac5f30d3a0a28c754f.pdf","moq":"N/A","lead_time":"1工作日","price_tiers":"5+ ￥1.3624 | 50+ ￥1.0786 | 150+ ￥0.9302 | 500+ ￥0.7597 | 3000+ ￥0.7269 | 6000+ ￥0.7074","hk_price":"N/A","mainland_price":"5+ ￥1.3624 | 50+ ￥1.0786 | 150+ ￥0.9302 | 500+ ￥0.7597 | 3000+ ￥0.7269 | 6000+ ￥0.7074","query_model":"TS5A3159DCKR"}`
